# PreventSleep

Windows PC のスリープ・画面消灯を防止し、設定ファイルに従ってウィンドウを自動再配置するユーティリティです。

## バージョン

v2.5.0 (Rust 実装)

## 機能

- **スリープ・画面消灯防止** — Windows API でディスプレイの電源オフを抑止し、30秒ごとにマウス微小移動を送信
- **ウィンドウ自動再配置** — `PreventSleep.txt` の設定に従い、起動時・モニター構成変化時にウィンドウを配置
- **マルチモニター対応** — モニター数変化を検知して自動的に再配置を実行
- **縦長高解像度モニターの仮想分割** — 縦長かつ高さが 1500px 超のモニターは、上下2画面として扱って配置対象化
- **安全マージン付き配置** — 各モニター作業領域の内側 (左=3px, 右=3px, 上=5px, 下=0px) を有効表示領域として画面端張り付きを回避
- **原点モニター基準フォールバック** — 配置先が不明なウィンドウは `(0,0)` 基準のモニターへ補正配置
- **モニター電源ON検知** — モニターが復帰した際に 2 秒後に再配置を実行
- **グローバルホットキー** — アプリが非アクティブでも `Ctrl+Alt+Z` で配置適用、`Ctrl+Alt+X` で階段配置を実行

## ビルド

### 必要な環境

```
rustup (https://rustup.rs)
Visual Studio Build Tools 2019 以降 (MSVC ツールチェーン)
```

### セットアップ

```powershell
# Rust インストール (未インストールの場合)
winget install Rustlang.Rustup

# MSVC ターゲットを追加 (通常は自動)
rustup target add x86_64-pc-windows-msvc
```

### ビルド

- ローカル環境では、debugビルドを行う。

```powershell
cd c:\path\to\PreventSleep
cargo build
```

生成物: `target\debug\PreventSleep.exe`

## 使い方

### GUI モード

```
PreventSleep.exe
PreventSleep.exe noprevent   # スリープ防止を無効にして起動
```

| ボタン | 動作 |
|---|---|
| 配置適用 | PreventSleep.txt に従ってウィンドウを再配置 |
| 階段配置 | ルール不一致ウィンドウを階段状に再配置 |
| １画面配置 | 原点モニターのみ存在する前提で再配置 |
| ウィンドウ一覧 | 現在の可視ウィンドウ一覧をログに表示 |
| モニターOFF | スリープ防止を解除してモニターをオフにする |

### グローバルホットキー

アプリが非アクティブ（他アプリ操作中）でも動作します。

| キー | 動作 |
|---|---|
| Ctrl + Alt + Z | 配置適用（PreventSleep.txt に従って全ウィンドウを再配置） |
| Ctrl + Alt + X | 階段配置（ルール不一致ウィンドウを階段状に並べる） |

### 再配置ルール（要点）

- 配置計算はモニター作業領域（タスクバー除外）を使用
- 縦長かつ高さが 1500px 超のモニターは、内部的に上下2つの仮想モニターとして扱う
- 画面内補正は (左=3px, 右=3px, 上=5px, 下=0px) の内側領域に対して実施
- ルール未一致ウィンドウの `階段配置` は、**モニターごと** に階段カーソルを持って `(+35,+35)` で進行
- ルール一致/不一致を問わず、左上座標がどのモニターにも属さない場合は原点モニター基準で補正

### CLI モード

```
PreventSleep.exe set         # ウィンドウを再配置して終了
PreventSleep.exe monitoroff  # モニターをオフにして終了
```

## 設定ファイル (`PreventSleep.txt`)

アプリケーションと同じディレクトリに置く CSV ファイル。

### 書式

```
<タイトル正規表現>,<クラス名正規表現>,<X>,<Y>,<幅>,<高さ>[,<画面数]
```

| フィールド | 説明 |
|---|---|
| タイトル正規表現 | ウィンドウタイトルのマッチ正規表現。空欄は任意 |
| クラス名正規表現 | ウィンドウクラス名のマッチ正規表現。空欄は任意 |
| X / Y | 配置先の左上座標。`100` のようなピクセル指定に加え、`10%` のような割合指定も可 |
| 幅 / 高さ | ウィンドウサイズ (ピクセル)。`*` 指定で対象画面の有効表示領域いっぱい、`70%` などの割合指定も可 |
| 画面数 | 接続画面数フィルタまたは `@N` による配置対象モニター指定。省略すると `12345` |

#### 画面数の `@N` 指定

- `@1` : 1画面目（原点モニタ）
- `@2` : 2画面目があれば2画面目、なければ1画面目
- `@3` : 3画面目があれば3画面目、なければ2→1の順でフォールバック

`@N` が含まれる行は、実装上は接続画面数フィルタを行わず、配置対象モニター指定として扱います。
`@N` 指定時は、対象画面の有効表示領域を基準に画面内へ収まるよう補正されます。
画面サイズの表示ログ（`# 1, ...`）も、この `@N` 順で表示されます。

#### 幅・高さの `*` 指定

- 幅または高さに `*` を指定すると、左上座標（設定値・階段配置等で決まった値）を維持し、
	右端/下端を表示先画面の有効表示領域右端/下端に合わせるようにサイズを決定します。
- 最大化ではなく、通常ウィンドウのサイズ変更として配置します。

#### 幅・高さの `%` 指定

- 幅または高さに `70%` のような割合を指定すると、左上座標を基準にした残り領域に対する比率としてサイズを決めます。
- たとえば `*, 70%` の場合、幅は有効表示領域の右端までの残り幅いっぱい、
  高さは `(Ymax - Y) × 0.7` になります。

#### X・Y の `%` 指定

- X または Y に `10%` のような割合を指定すると、配置対象ディスプレイの有効表示領域に対する相対位置として解釈します。
- 算出式は以下です。
	- `X = left + (effective_width × p)`
	- `Y = top + (effective_height × p)`
	- ここで `p = 10%` なら `0.1`
- `@N` 指定時は `@N` で選ばれたディスプレイを基準に計算します。
- `@N` 未指定時は、そのウィンドウが現在存在するディスプレイ（判定できない場合は原点ディスプレイ）を基準に計算します。

#### `@N` 指定時の X・Y ピクセル値

- `@N` を指定した行で X/Y をピクセル値（例: `0`, `100`）で書いた場合、
  その値は **対象ディスプレイ有効表示領域の左上からのオフセット** として扱います。
- 例: `..., 0, 0, *, 90%, @3` は「3画面目の有効表示領域の左上」を基準に配置します。

### 記述例

```
# コメント行 (# で始まる行は無視)
####,####,0,0,0,0          # 区切り行 (#### で始まる行も無視)

Notepad,,100,100,800,600   # タイトルに "Notepad" を含むウィンドウを (100,100) に配置
,CabinetWClass,0,0,1280,720,12  # クラス名が CabinetWClass のウィンドウ (1〜2画面時)
.*Edge$,Chrome_WidgetWin_1,0,5,*,*,@1  # Edge を原点モニタ有効領域いっぱいに配置
Notepad,,10%,10%,70%,70%,@2  # 2画面目の有効表示領域の10%位置に、70%サイズで配置
```
