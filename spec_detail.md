## 配置適用・階段配置の詳細仕様（改訂）

本書は、`配置適用` と `階段配置` の挙動を実装者・テスターが同じ解釈になるように明文化した仕様である。

### 1. 共通定義

- **モニタ範囲** は Windows の作業領域（タスクバー等を除く領域）を用いる。
- **仮想モニタ分割**: `abs_height > abs_width` かつ `abs_height > 1500` を満たすモニタは、実装上、上下2つの仮想モニタに分割して扱う。
  - 分割基準はタスクバーを含む絶対領域（`rcMonitor` / `abs_*`）の中央とする。
  - 上側: `(left, top)-(right, split_work_y)`
  - 下側: `(left, split_work_y)-(right, bottom)`
  - `split_work_y` は絶対領域中央を作業領域へ射影し、作業領域高さが確保できる範囲へクランプする。
  - 以降の `num_display`・画面番号付け（`@N`）・所属判定は、この分割後モニタ集合を基準とする。
- **モニタ情報ログ** は、作業領域（work）と絶対領域（abs）の両方を出力する。
- **有効表示領域** は、モニタ範囲から固定マージンを引いた内側領域とする。
  - `WINDOW_MARGIN_LEFT = 3`（定数：左）
  - `WINDOW_MARGIN_RIGHT = 3`（定数：右）
  - `WINDOW_MARGIN_TOP = 5`（定数：上）
  - `WINDOW_MARGIN_BOTTOM = 0`（定数：下）
  - 例: モニタ範囲が `(0,0)-(1920,1080)` の場合、有効表示領域は `(3,5)-(1917,1080)`
- **階段オフセット** は `CASCADE_OFFSET = 35`（定数）
- **座標所属判定** は左上座標 `(x,y)` が `left <= x < right` かつ `top <= y < bottom` を満たすモニタとする。
- **原点モニタ** は次の優先順で決定する。
  1. 左上が `(0,0)` のモニタ
  2. `(0,0)` を内包するモニタ
  3. `|left| + |top|` が最小のモニタ

### 2. 処理対象ウィンドウ

- 列挙されたトップレベル可視ウィンドウを対象とする。
- ただし以下は対象外とする。
  - PreventSleep 自身
  - 最小化・最大化状態のウィンドウ
  - タイトル空文字のウィンドウ
  - 下記条件に該当するウィンドウ

```
(title == "Program Manager" && class_name == "Progman")
        || class_name == "Shell_TrayWnd"
        || class_name == "Shell_SecondaryTrayWnd"
        || class_name == "Windows.UI.Core.CoreWindow"
```

- PreventSleep 自身は別処理として、最終的に **原点モニタ左下（作業領域内）** に配置する。

### 3. ルールマッチ（PreventSleep.txt）

- マッチ条件は以下の AND。
  - `displays` 判定に通ること
  - タイトル正規表現（空欄は無条件一致）
  - クラス名正規表現（空欄は無条件一致）
- `displays` 判定は実装上、次の順で評価する。
  1. 空文字（空欄）なら通す
  2. `@N` を含むなら通す（`@N` は接続画面数フィルタではなく配置対象モニタ指定として扱う）
  3. それ以外は、現在接続画面数 `num_display` が `displays` に含まれる場合のみ通す
- 同一ウィンドウに複数ルールがマッチした場合は、**ファイル後勝ち（最後にマッチしたルール）** を適用する。

#### 3.1 `@N` の指定位置（display フィールド）

- 例: `...,X,Y,W,H,@2` のように書く。
- 画面番号の定義:
  1. `@1` = 原点モニタ
  2. `@2, @3, ...` = 原点モニタを除く残りモニタを以下の順で並べた番号
     - まず `X > 0` のモニタを、原点 `(0,0)` からのユークリッド距離が短い順に並べる
     - 次に `X < 0` のモニタを、同じくユークリッド距離が短い順に並べる
     - `X = 0` かつ原点ではないモニタは、実装上は上記の後ろに距離順で並べる
- フォールバック:
  - `@2` で2画面目がない場合は1画面目
  - `@3` で3画面目がない場合は2→1の順で最終的に存在する番号へフォールバック
- `@N` 指定時は、その画面の有効表示領域を使って `X/Y/W/H` を補正する。
- 画面サイズの表示ログ（`# 1, ...`）もこの `@N` 順に並ぶ。
- ログ書式は `# n, work=(left, top, width, height), abs=(left, top, width, height)`。

#### 3.1.1 `@N` の実適用タイミング

- `@N` はルールマッチ条件ではなく、ルール一致後の「配置先モニタ決定」で使う。
- `@N` で選ばれたモニタは以下に使われる。
  - X/Y が `%` の場合の基準有効表示領域
  - X/Y がピクセル値の場合の基準オフセット原点（有効表示領域左上）
  - 幅/高さ（`*` / `%`）の算出基準有効表示領域
  - 最終クランプ先モニタ

#### 3.2 幅・高さフィールド拡張（`*` / `%` / '@' ）

- 幅または高さに `*` を指定した場合、左上座標（設定値・階段配置等で決まった値）を維持し、
  右端/下端を表示先画面の有効表示領域右端/下端に合わせる。
- 幅または高さに `70@` のような `@` 指定をした場合、配置対象モニタの有効表示領域を基準に、その割合でサイズを決める。
- 例: `*, 70@` なら幅は最大、 高さは `Ymax × 0.7` とする。
- 幅または高さに `70%` のような `%` 指定をした場合、左上座標から右端/下端までの残り領域を基準に、その割合でサイズを決める。
- 例: `*, 70%` なら幅は最大、 高さは `(Ymax - Y) × 0.7` とする。
- `%` 指定は 0 未満に丸めず、負値は 0 として扱う。

#### 3.3 X・Y フィールドの `%` 指定

- X または Y に `10%` のような `%` 指定をした場合、配置対象モニタの有効表示領域を基準に座標を算出する。
- 算出式:
  - `X = effective.left + (effective.width × p)`
  - `Y = effective.top + (effective.height × p)`
  - `p = 10%` なら `0.1`
- `@N` 指定時は、`@N` で選ばれたモニタの有効表示領域を基準に算出する。
- `@N` 未指定時は、対象ウィンドウの現在左上が属するモニタ（不明時は原点モニタ）を基準に算出する。
- 算出後は 4章の補正規則に従って最終配置を確定する。

#### 3.4 X・Y フィールドの `*` 指定

- X または Y に `*` を指定した場合、そのルールは **対象モニタでの階段配置** として扱う。
- 配置先モニタは以下の順で決める。
  1. `@N` があれば `@N` で選ばれたモニタ
  2. なければ対象ウィンドウの現在左上が属するモニタ
  3. それもなければ原点モニタ
- 当該モニタの階段カーソル位置を `(left, top)` として採用する。
- 階段カーソル初期値は、そのモニタの有効表示領域左上とする。
- 同一モニタに対する次の階段配置位置は `(+35,+35)` 進める。
- 幅/高さは、解決後の `(left, top)` を基準に `*` / `%` / px を通常どおり算出する。

### 4. サイズ・座標補正の原則

- 幅・高さが 1 未満になった場合は 1 に丸める。
- 配置時は対象モニタの **有効表示領域内** に左上・右下が収まるよう補正する。
- 補正優先順は以下とする。
  1. 可能な限りサイズを維持して左上座標を補正
  2. それでも収まらない場合のみサイズを縮小

### 5. ルール一致ウィンドウの挙動（実装上の判断順）

- ルールファイルを先頭から走査し、マッチしたルールで都度候補値を更新する（最後にマッチしたルールが有効）。
- マッチした各ルールについて、以下の順で算出する。
  1. `displays` から `@N` を抽出できれば、そのモニタを `forced_target_monitor` とする
  2. `%` 座標時のフォールバック基準として、対象ウィンドウの現在左上が属するモニタ（なければ原点）を決める
  3. X/Y に `*` が含まれる場合は、対象モニタの階段カーソル位置を `(left, top)` に採用する
  4. それ以外では X/Y の基準モニタを決める
     - `@N` あり: `forced_target_monitor`
     - `@N` なし かつ X/Y が両方ピクセル: `(X,Y)` が属するモニタ（なければ 2 のフォールバック）
     - それ以外（X または Y が `%`）: 2 のフォールバック
  5. 4 の有効表示領域に対して X/Y を解決する（`%` は相対座標計算）
  6. 幅/高さ算出用モニタを決める
     - `@N` あり: `forced_target_monitor`
    - それ以外: 解決後 `(left,top)` が属するモニタ（なければ 4 の基準モニタ）
  7. 6 の有効表示領域に対し、残り領域 `right-left`, `bottom-top` を使って `W/H`（`*` / `%` / px）を解決する
  8. 階段配置ルールが最終採用された場合、そのモニタの階段カーソルを `(+35,+35)` 進める
  9. 最後にマッチしたルールの `(left, top, width, height)` を最終候補として保持する

### 6. ルール不一致ウィンドウ（配置適用）

- 現在左上が属するモニタがある場合
  - 現在位置・現在サイズを初期値にする。
  - 4章の原則で有効表示領域内に収める。
- 現在左上がどのモニタにも属さない場合
  - 現在位置・現在サイズを初期値にしたまま、原点モニタを補正先として 4章の原則で収める。
  - このため、必ずしも原点モニタ左上に揃うとは限らず、右端/下端側に寄って収まる場合がある。

### 7. ルール不一致ウィンドウ（階段配置）

- 各モニタごとに階段カーソルを持つ。
  - 初期値: そのモニタの有効表示領域左上
- 対象ウィンドウの割当モニタ
  - 現在左上が属するモニタがあればそのモニタ
  - なければ原点モニタ
- 配置手順
  1. 割当モニタの階段カーソル位置を左上初期値として配置
  2. 4章の原則で有効表示領域内に収める
  3. そのモニタの階段カーソルを `(+35,+35)` 進める
  4. 当該ウィンドウを前面化する（常時最前面化はしない）

### 8. 境界ケース

- 有効表示領域が極端に小さい場合でも、最終サイズは最小 `1x1` を保証する。
- モニタ情報を取得できない場合は、仮想モニタ `(0,0)-(1920,1080)` を用いて同様に処理する。
- ルールファイルが存在しない/読めない場合は「全ウィンドウがルール不一致」として扱う。
