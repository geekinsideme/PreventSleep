## 配置適用・階段配置の詳細仕様（改訂）

本書は、`配置適用` と `階段配置` の挙動を実装者・テスターが同じ解釈になるように明文化した仕様である。

### 1. 共通定義

- **モニタ範囲** は Windows の作業領域（タスクバー等を除く領域）を用いる。
- **有効表示領域** は、モニタ範囲から固定マージンを引いた内側領域とする。
  - `WINDOW_MARGIN_LEFT = 3`（定数：左）
  - `WINDOW_MARGIN_RIGHT = 3`（定数：右）
  - `WINDOW_MARGIN_TOP = 5`（定数：上）
  - `WINDOW_MARGIN_BOTTOM = 0`（定数：下）
  - 例: モニタ範囲が `(0,0)-(1920,1080)` の場合、有効表示領域は `(3,5)-(1917,1080)`
- **階段オフセット** は `CASCADE_OFFSET = 35`（定数）
- **座標所属判定** は左上座標 `(x,y)` が `left <= x < right` かつ `top <= y < bottom` を満たすモニタとする。
- **原点モニタ** は次の優先順で決定する。
  1. 左上が `(0,0)` のモニタ
  2. `(0,0)` を内包するモニタ
  3. `|left| + |top|` が最小のモニタ

### 2. 処理対象ウィンドウ

- 列挙されたトップレベル可視ウィンドウを対象とする。
- ただし以下は対象外とする。
  - PreventSleep 自身
  - 最小化・最大化状態のウィンドウ
  - タイトル空文字のウィンドウ
  - 下記条件に該当するウィンドウ

```
(title == "Program Manager" && class_name == "Progman")
        || class_name == "Shell_TrayWnd"
        || class_name == "Shell_SecondaryTrayWnd"
        || class_name == "Windows.UI.Core.CoreWindow"
```

- PreventSleep 自身は別処理として、最終的に **原点モニタ左下（作業領域内）** に配置する。

### 3. ルールマッチ（PreventSleep.txt）

- マッチ条件は以下の AND。
  - タイトル正規表現（空欄は無条件一致）
  - クラス名正規表現（空欄は無条件一致）
  - 現在モニタ数が displays 条件に含まれること
- 同一ウィンドウに複数ルールがマッチした場合は、**ファイル後勝ち（最後にマッチしたルール）** を適用する。

#### 3.1 X フィールド拡張（`@N`）

- X に `@N` を指定した場合、**N画面目** を配置対象モニタとして扱う。
- 画面番号の定義:
  1. 1画面目 = 原点モニタ
  2. 2画面目以降 = 列挙された残りモニタ順
- フォールバック:
  - `@2` で2画面目がない場合は1画面目
  - `@3` で3画面目がない場合は2→1の順で最終的に存在する最大番号へフォールバック
- `@N` 指定時の初期Xは対象モニタの有効表示領域左端を用いる。

#### 3.2 幅・高さフィールド拡張（`*`）

- 幅または高さに `*` を指定した場合、左上座標（設定値・階段配置等で決まった値）を基準に、
  右下座標の決定時に配置先モニタの有効表示領域の右端/下端を用いる。
- これは最大化ではなく、通常の位置・サイズ設定として適用する。

### 4. サイズ・座標補正の原則

- 幅・高さが 1 未満になった場合は 1 に丸める。
- 配置時は対象モニタの **有効表示領域内** に左上・右下が収まるよう補正する。
- 補正優先順は以下とする。
  1. 可能な限りサイズを維持して左上座標を補正
  2. それでも収まらない場合のみサイズを縮小

### 5. ルール一致ウィンドウの挙動

- 指定 `(x,y,w,h)` を初期値として配置する。
- `(x,y)` がどのモニタにも属さない場合、**原点モニタの有効表示領域左上** を基準位置とする。
- その後、4章の原則で有効表示領域内に収める。

### 6. ルール不一致ウィンドウ（配置適用）

- 現在左上が属するモニタがある場合
  - 現在位置・現在サイズを初期値にする。
  - 4章の原則で有効表示領域内に収める。
- 現在左上がどのモニタにも属さない場合
  - 原点モニタの有効表示領域左上を初期位置にする。
  - 現在サイズを初期サイズとして、4章の原則で収める。

### 7. ルール不一致ウィンドウ（階段配置）

- 各モニタごとに階段カーソルを持つ。
  - 初期値: そのモニタの有効表示領域左上
- 対象ウィンドウの割当モニタ
  - 現在左上が属するモニタがあればそのモニタ
  - なければ原点モニタ
- 配置手順
  1. 割当モニタの階段カーソル位置を左上初期値として配置
  2. 4章の原則で有効表示領域内に収める
  3. そのモニタの階段カーソルを `(+35,+35)` 進める
  4. 当該ウィンドウを前面化する（常時最前面化はしない）

### 8. 境界ケース

- 有効表示領域が極端に小さい場合でも、最終サイズは最小 `1x1` を保証する。
- モニタ情報を取得できない場合は、仮想モニタ `(0,0)-(1920,1080)` を用いて同様に処理する。
- ルールファイルが存在しない/読めない場合は「全ウィンドウがルール不一致」として扱う。
